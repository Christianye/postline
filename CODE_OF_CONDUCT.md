# Code of Conduct

postline is a small project run by a small team. The rules are short.

## What we expect

- **Be decent.** Disagree about code freely; disagree about people never.
- **Stay on topic.** This repo is about a Feishu/Lark bot framework. Issues, PRs, and discussions should concern the code or its use.
- **Assume good faith.** Maintainers and contributors give time for free. A terse review is not a hostile review.
- **English or 中文 both welcome** in issues and discussions. PR titles, commit messages, and code comments stay English.

## What we won't tolerate

Harassment, personal attacks, or conduct that makes others unsafe participating. Scope includes issues, PRs, discussions, and any postline-branded space.

## Reporting

If someone's behaviour crosses the line, report it **privately** through [GitHub's Report Abuse flow](https://docs.github.com/en/communities/maintaining-your-safety-on-github/reporting-abuse-or-spam) or by opening a [private security advisory](https://github.com/Christianye/postline/security/advisories/new) on this repo. The advisory channel is monitored by the same people who handle `SECURITY.md` reports.

Expect acknowledgement within 5 business days.

## Consequences

Maintainers may remove comments, close threads, or block accounts at their discretion. No public tribunals — decisions are made privately and communicated to the affected party.

## Scope

This applies to anyone participating in postline spaces, including maintainers. There is no separate rulebook for maintainers; the same expectations apply.
